
more important features:
- user login
- admin user management
- user html templates are stored in files in a directory separate from the code repo
- data is stored in MySQL database
- application admin database user can create/modify/drop tables
- normal database user can only insert/update/delete records
- tree permission structure (use either json or level1.level2.level3 system)


less important features:
- json api




automated form generator:
- list without controls or with read-only controls
- justified (multi-column) with editable controls for single record insert/update
- justified (multi-column) either without controls or with read-only controls for confirmation of deletion (???)

- include option to generate template files (which can then be customised further)
